# Internet Health Monitor — Agent Instructions

## Codegen: countries.yaml is the source of truth

`config/countries.yaml` drives everything. When it changes, **four categories of files** must be regenerated via `just generate`:

1. `assets/ingestion/{ipv6,https,dnssec_validation,roa_4,roa_6}/*.asset.yml` — one per country per metric
2. `assets/transformations/{ipv6,https,dnssec_validation}_combined.sql` and `roa_combined.sql` — UNION ALL per country
3. `pipeline.yml` — countries variable list
4. `tests/conftest.py` — **auto-generated by `scripts/generate_assets.py` from `scripts/conftest.tpl`; edit the `.tpl`, not the generated file**

To add/remove countries:
```bash
just add-country CC [CC ...]    # adds to countries.yaml + regenerates everything
just remove-country CC [CC]     # removes + regenerates
just check-drift                # CI check: validates ingestion assets only (not pipeline.yml or conftest.py)
```

## Commands

```bash
just ci              # full CI suite (lint → typecheck → format → test → validate → drift-check)
just generate        # regenerate all pipeline assets from countries.yaml
just dashboard       # launch Dash app → http://localhost:8050
just run-pipeline    # run Bruin pipeline (requires ISOC_PULSE_TOKEN)
just clean-db        # delete data/internet_health.db
just test-cov        # pytest with coverage (covers dashboard/ package)
just docker-up       # start dashboard in Docker (detached)
```

## CI checks — order and scope

`just ci` runs these in order. All must pass:

1. `just install` — `uv sync --group dev`
2. `just lint` — `ruff check .`
3. `just typecheck` — `pyrefly check dashboard/ tests/ assets/enrichment/` (only these three directories, not the whole repo)
4. `just format` — `ruff format .` (**writes** formatting; CI uses `ruff format --check .`)
5. `just test` — `pytest tests/ -v`
6. `just validate` — `bruin validate .` (requires ISOC_PULSE_TOKEN)
7. `just check-drift` — verifies generated files match countries.yaml

After editing code, run `just ci` to verify.

## Data architecture

Medallion layers:

- **Bronze (`raw.*)`**: One table per ingestion asset. `delete+insert` materialization with `incremental_key: date`.
- **Silver (`staging.*)`**: `*_combined` tables UNION ALL countries. Transform assets use `create+replace`.
- **Gold (`marts.*)`**: `internet_health_summary`, `country_rankings`, `health_scoring`.

Key pipeline logic that's not obvious from filenames:

- **IPv6 is monthly** and drives the summary JOIN in `internet_health_summary.sql`. HTTPS, DNSSEC, and ROA are daily metrics aggregated to monthly via `DATE_TRUNC('month', date) + AVG()` before joining.
- **Trend lines preserve daily granularity** — `get_daily_metric_timeseries()` queries `staging.*_combined` tables directly, not `marts.internet_health_summary`.
- **ROA merges IPv4+IPv6** in `roa_combined.sql`: `(ipv4_value + ipv6_value) / 2 * 100 AS roa_score`.
- **All scores are 0–100 scale**. The `value * 100` conversion happens in the silver SQL transforms.
- **Health score** = `(ipv6 + https + dnssec + roa) × 0.25` — equal weight, four metrics. Computed in `country_rankings.sql` with `COALESCE(metric, 0)`.

## Testing

- Tests are **self-contained** — `conftest.py` creates session-scoped temp DuckDB instances with deterministic data. No need to run `just run-pipeline` or have a real database before `just test`.
- `conftest.py` is auto-generated by `scripts/generate_assets.py` from `scripts/conftest.tpl`. Edit the `.tpl`, not the generated file.

## Entrypoints

- **Dashboard**: `uv run python -m dashboard.app` → `http://localhost:8050`
- **Pipeline**: `bruin run . --start-date <date>` (justfile defaults to `2026-03-01`)
- **Package layout**: `dashboard/data/` (DuckDB queries), `dashboard/layouts/` (Dash page layouts), `dashboard/components/` (reusable UI), `assets/enrichment/` (Bruin Python assets)

## Operational gotchas

- **`ISOC_PULSE_TOKEN`** env var is required for `just run-pipeline` and `just validate`. The justfile loads `.env` via `set dotenv-load`.
- **DuckDB** lives at `data/internet_health.db` (gitignored). If tests or dashboard say "database not found", run `just run-pipeline` first.
- **`just run-pipeline`** uses `start_date := "2026-03-01"` from the justfile, which may differ from `start_date` in `pipeline.yml`.
- **`health_scoring.py`** hardcodes `duckdb.connect("data/internet_health.db")` — relative to project root.
- **`uv.lock`** is committed to git — run `uv lock` after any dependency change in `pyproject.toml`.

## Code style

- ruff: line-length 100, double quotes, rules `E,F,W,I,N,UP,B,C4` (E501 ignored)
- Python 3.12 (enforced via `.python-version`)
- No comments unless explicitly requested
- **pyrefly suppressions**: `# pyrefly: ignore[error-kind]` goes **on the error line itself**, not on the function signature. Plotly's dynamic layout attributes (`fig.layout.annotations`, `fig.layout.xaxis`, `fig.layout.shapes`, etc.) require `#[missing-attribute]` suppressions in test files.
