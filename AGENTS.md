# Internet Health Monitor ‚Äî Agent Instructions

## Codegen: countries.yaml is the source of truth

`config/countries.yaml` drives everything. When it changes, **four categories of files** must be regenerated via `just generate`:

1. `assets/ingestion/{ipv6,https,dnssec_validation,roa_4,roa_6}/*.asset.yml` ‚Äî one per country per metric
2. `assets/transformations/{ipv6,https,dnssec_validation}_combined.sql` and `roa_combined.sql` ‚Äî UNION ALL per country
3. `pipeline.yml` ‚Äî countries variable list
4. `tests/conftest.py` ‚Äî **auto-generated, never edit manually**

To add/remove countries:
```bash
just add-country CC [CC ...]    # adds to countries.yaml + regenerates everything
just remove-country CC [CC]     # removes + regenerates
just check-drift                # CI check: exits 1 if generated files are stale
```

## Commands

```bash
just ci              # full CI suite (lint ‚Üí typecheck ‚Üí format ‚Üí test ‚Üí validate ‚Üí drift-check)
just generate        # regenerate all pipeline assets from countries.yaml
just dashboard       # launch Dash app ‚Üí http://localhost:8050
just run-pipeline    # run Bruin pipeline (requires ISOC_PULSE_TOKEN)
just clean-db        # delete data/internet_health.db
just test-cov        # pytest with coverage (covers dashboard/ package)
just docker-up       # start dashboard in Docker (detached)
```

## CI checks ‚Äî order and scope

`just ci` runs these in order. All must pass:

1. `just install` ‚Äî `uv sync --extra dev`
2. `just lint` ‚Äî `ruff check .`
3. `just typecheck` ‚Äî `ty check dashboard/ tests/ assets/enrichment/` (only these three directories, not the whole repo)
4. `just format` ‚Äî `ruff format .` (**writes** formatting; CI uses `ruff format --check .`)
5. `just test` ‚Äî `pytest tests/ -v`
6. `just validate` ‚Äî `bruin validate .` (requires ISOC_PULSE_TOKEN)
7. `just check-drift` ‚Äî verifies generated files match countries.yaml

After editing code, run `just ci` to verify.

## Data architecture

Medallion layers:

- **Bronze (`raw.*)`**: One table per ingestion asset. `delete+insert` materialization with `incremental_key: date`.
- **Silver (`staging.*)`**: `*_combined` tables UNION ALL countries. Transform assets use `create+replace`.
- **Gold (`marts.*)`**: `internet_health_summary`, `country_rankings`, `health_scoring`.

Key pipeline logic that's not obvious from filenames:

- **IPv6 is monthly** and drives the summary JOIN in `internet_health_summary.sql`. HTTPS, DNSSEC, and ROA are daily metrics aggregated to monthly via `DATE_TRUNC('month', date) + AVG()` before joining.
- **Trend lines preserve daily granularity** ‚Äî `get_daily_metric_timeseries()` queries `staging.*_combined` tables directly, not `marts.internet_health_summary`.
- **ROA merges IPv4+IPv6** in `roa_combined.sql`: `(ipv4_value + ipv6_value) / 2 * 100 AS roa_score`.
- **All scores are 0‚Äì100 scale**. The `value * 100` conversion happens in the silver SQL transforms.
- **Health score** = `(ipv6 + https + dnssec + roa) √ó 0.25` ‚Äî equal weight, four metrics. Computed in `country_rankings.sql` with `COALESCE(metric, 0)`.

## Testing

- Tests are **self-contained** ‚Äî `conftest.py` creates session-scoped temp DuckDB instances with deterministic data. No need to run `just run-pipeline` or have a real database before `just test`.
- `conftest.py` is auto-generated by `scripts/generate_assets.py`. Edits will be overwritten on next `just generate`.

## Operational gotchas

- **`ISOC_PULSE_TOKEN`** env var is required for `just run-pipeline` and `just validate`. The justfile loads `.env` via `set dotenv-load`.
- **DuckDB** lives at `data/internet_health.db` (gitignored). If tests or dashboard say "database not found", run `just run-pipeline` first.
- **`just run-pipeline`** uses `start_date := "2026-03-01"` from the justfile, which may differ from `start_date` in `pipeline.yml`.
- **`health_scoring.py`** hardcodes `duckdb.connect("data/internet_health.db")` ‚Äî relative to project root.
- **`uv.lock`** is committed to git ‚Äî run `uv lock` after any dependency change in `pyproject.toml`.
